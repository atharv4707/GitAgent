# Mindset Analysis Skill

Reveal the thinking behind the code by analyzing developer notes, commit message quality, and language signals that indicate shortcuts or avoidance.

## Language Signals

- "later" or "eventually": Procrastination, technical debt accumulation
- "for now" or "quick fix": Temporary solutions becoming permanent
- "I think" or "maybe": Uncertainty without validation
- "fine" or "works somehow": Settling below quality threshold

## Commit Patterns

- Vague messages: Poor communication habits
- Missing context: Avoiding documentation
- Short commits: Rushing or carelessness

## Avoidance Patterns

- Security shortcuts: "security can wait" mentality
- Missing tests: Avoiding verification
- No error handling: Optimistic coding bias

## Output Format

```
MINDSET ANALYSIS
----------------------------------------
[Core thinking pattern identified]
[Consequence of this thinking]
[What developer is likely thinking]
[Reality check]
```
